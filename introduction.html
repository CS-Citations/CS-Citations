Computer Science

In order for you to write programs to control a computer, you should first know how a computer works,
As it will make a lot of things easier for you.

1) Programming in it's essence

Programming is the process of planning, designing, and writing a set of instructions for a computer to execute.

There are primarily only 4 kinds of instructions that a computer can perform :

  1. Reading data from Memory.

  2. Comparing 2 or more values.

  3. Performing Mathematical operations on 1 or more values.

  4. Writing data to Memory.

And that's it,
Every operation in a computer is a combination of these 4 instructions.

2) Components of a Computer

Computers consist of a number of components, all working together to form a logical machine.

As a programmer, there are only 3 parts that you should keep in mind :

  1. CPU

  2. Memory

  3. I/O Devices

The Central Processing Unit performs the computations and calculations.
Memory stores the Data & Instructions for the CPU.
I/O devices allow the user to send and receive data from Memory and view them.

3) How Computers store Data

Computers are electronic devices, they work by controlling the flow of electricity.
So, how do they store Data? The answer is binary, a number system with only two digits (0 and 1).

In the context of Computers :

  • A High-Voltage signal means 1

  • A Low-Voltage signal means 0

Each High and Low Voltage is stored into Memory using Transistors*

For example, we can assign the following combinations of 0's and 1's as:

  • 1001000 -> H

  • 1100101 -> e

  • 1101100 -> l

  • 1101100 -> l

  • 1101111 -> o

To store the word 'Hello' into Memory, using High and Low Voltages (Binary).

The above Binary combinations are according to the ASCII encoding system.
There are other standards as well, such as UTF-8, Unicode, etc.

A single (1 or 0) is called a Bit.
8 bits make a Byte.
1024 bytes make a MegaByte
1024 megabytes make a GigaByte
and so on.

4) Programming Languages

Programming Languages allow you to execute instructions, by writing code in a human-readable format, without having to worry about the underlying complicated process.

Programming Languages are commonly classified as :

  1. Compiled :

    2. Like C, C++, C#, Go, Fortran, Swift, etc

  2. Interpreted

    3. Like Python, JavaScript, R, Ruby, etc

( Although theoretically, any language can be Compiled or Interpreted or both )

The main difference being that :
Compiled Languages are translated/compiled into Binary code.
Interpreted Languages run directly on a program, called an Interpreter.

( Compiled Languages are usually significantly faster than Interpreted ones )

Coding Tips

( Assuming that you know some basic programming concepts )

  1. If a bunch of lines work together to perform a task, make it a Function.

  2. Each Function should exist to only do 1 thing, and not multiple.

  3. Your syntax should have sufficient indentation, 1-Tab for every Function, Class, Loop, and If Statement.

  4. The Name of your Variables, Functions, and Classes should be self-explanatory.

  5. If your code is getting too long(>200 lines), try dividing it into multiple files if you can.

  6. Prompts given to the user for input should be clear and to the point.

  7. Consider every single invalid value while taking Inputs from the user.

  8. Never look at answers for coding questions on Quora.

  9. Memorizing Syntax doesn't make you a better programmer, your problem-solving skills do.

Programming is a process that takes time and effort, a CS project can take anywhere between a Day and a Year, sometimes even longer.
Most if not all Large Open-Source projects started out as a personal endeavor. So be patient, keep coding, and never abandon your projects.
