Project Guide

So you have an idea for a project, could be a software, could be a video-game, a website, or even a mobile app. The only problem is that you don't know where to start.
So, here's a definitive step-by-step guide on how to bring your project ideas to life (as a beginner).

The guide assumes that you already know the basics of your language ( which you can do from here )

1) Have clear Expectations.

Write down what you expect your program will be able to do, in order for it to function as expected.

For-example

Let's assume that you're making a To-Do list app with C++.
The above program is expected to:

  • Draw stuff on a window.

  • Create new notes and save them.

  • load previously saved notes and display them.

  • update and save old notes.

2) Research for Libraries(If needed)

Now go through your list(from Step 1) and one-by-one check if your language can natively perform that task. If it can't, google which module/library can do so.

For-example

For the prior example list(from Step 1) Creating, Saving and Loading notes (as a file) can be done with native C++.

But Drawing stuff in a window requires a library.
Search "C++ GUI Library" on Google, the easiest sources for finding stuff like this is StackOverflow & Reddit. ( Do not use Quora for coding related things)

Once you've found the library you were looking for, write it's name down.

3) How to learn a Library.

If you need to learn a new library, Search on YouTube Or StackOverflow to understand the basics, If you need more help, google stuff along the way.

You don't need to know everything, rather just enough to get the job done.
While reading documentation of a library is a good practice (if you want to know everything it has to offer), it's not completely necessary to do so.

4) Write Sudo-Code to make the process easier

Now that you've understood all the tools you'll need for your project. It's time to write some sudo-code.

Although it's not compulsory, writing sudo-code does help immensely when you're starting out as a beginner.

5) Implement the sudo-code into your project.

Don't write large chunks of code at once, even if you know exactly what you're doing.
Write sudo-code for 1 task and implement only that 1 task in your code at a time, don't rush things.

And if something breaks, comment out a single line of code, run the program, if the problem persists, un-comment that line and try a different one, google your errors if you need to.

Pro Tips

  1. If a bunch of lines work together to perform a task, make it a Function.

  2. Each Function should exist to only do 1 thing, and not multiple.

  3. Your syntax should have sufficient indentation, 1-Tab for every Function, Class, Loop, and If Statement.

  4. The Name of your Variables, Functions, and Classes should be self-explanatory.

  5. If your code is getting too long(>200 lines), try dividing it into multiple files if you can.

  6. Prompts given to the user for input should be clear and to the point.

  7. Consider every single invalid value while taking Inputs from the user.

  8. Never look at answers for coding questions on Quora.

  9. Memorizing Syntax doesn't make you a better programmer, your problem-solving skills do.

Programming is a process that takes time and effort, a CS project can take anywhere between a Day and a Year, sometimes even longer.
Most if not all Large Open-Source projects started out as a personal endeavor. So be patient, keep coding, and never abandon your projects.
